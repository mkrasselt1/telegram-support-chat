Features

Everything your support team needs

One JavaScript include. Full-featured chat. Replies land directly in Telegram.

📎

File & media sharing

Users can send images, PDFs, audio recordings and videos. Files are forwarded directly to Telegram with previews.

🎤

Voice messages

One-click audio recording via MediaRecorder API. Sent as native Telegram voice messages your agents can play instantly.

😀

Emoji picker

Built-in emoji picker with 400+ emoji across 6 categories. Zero external dependencies.

📍

Location sharing

Users can share their GPS location with one click. Appears as a native Telegram map pin — tap to open Maps.

🔔

Push notifications

HTML5 Notifications + Service Worker. Users get notified even when the tab is in the background.

🧵

Per-user Telegram threads

Every user gets their own forum topic in your Telegram group. Sessions survive page reloads via localStorage.

👤

User context payload

Pass logged-in user name, email and ID — agents see an info card with full context before the first message.

📧

Offline email alerts

If a user has been away for 5 minutes, an email notification fires when an agent replies — no message goes unseen.

⚡

Webhook mode

Register once with register_webhook.php. Telegram pushes replies instantly — no polling delay.

How it works

No daemon. No WebSockets.
Just PHP & Telegram.

The entire backend is triggered by client HTTP requests — no constantly running service needed on your server.

1

User opens the widget

A UUID session ID is stored in localStorage. PHP creates a Telegram forum topic for this session and sends an info card to your team.

2

Messages go to Telegram

Every text, file, location or voice message is forwarded to the correct Telegram thread in real time.

3

Agent replies in Telegram

Your team replies normally inside the Telegram thread. In webhook mode the reply reaches the widget in <1 second.

4

Widget polls for updates

Every 4 seconds the widget polls chat.php. New agent messages are read from persistent session history — no messages lost across tabs or return visits.

Integration

Two script tags. That's it.

ℹ️ The demo on this page uses endpoint: 'chat.php' pointing to the local PHP backend. Adjust the path to match your deployment. 
<!-- Place before </body> on every page -->
<script>
window.SupportChatConfig = {
  endpoint: '/support-chat/chat.php',   // required
  swPath  : '/sw.js',                  // service worker (must be at domain root)

  // Logged-in user info — shown to agents as context
  user: { name: 'Jane Doe', email: 'jane@example.com', id: 42 },

  // Theme — match your brand color
  theme: { primary: '#0088cc' },
};
</script>
<script src="/support-chat/support-chat.js" async></script>
